Calibration check (at k)

Before main grading, k outputs are double-rated to verify the rubric meets a pre-registered ICC threshold. Probabilities below assume true ICC = ρ.

i Calibration set size (k)

Number of outputs scored by both raters before main grading begins, to check that the rubric and raters produce reliable scores.

The proposal pre-registers k = 50.
i Strict threshold

If the sample ICC at k outputs meets or exceeds this, calibration passes outright and main grading proceeds.

The proposal sets this at 0.70 — the Koo & Li boundary between moderate and good agreement.
i Relaxed threshold

If the sample ICC falls between this value and the strict threshold, the bar is relaxed to this value, with explicit reporting in the paper.

Below this value, calibration fails and the rubric or raters need re-training.

The proposal sets this at 0.65.

i Rating effort

Each output is rated once per human, so M outputs × 2 raters gives the total session count.

Multiplied by the minutes-per-session figure from study setup (default 12), this gives the total rating hours, split evenly between the two raters.

i Joint-confidence mode

When you report several ICCs (3 rater pairs × D dimensions), each test carries its own error rate. Without adjustment, the chance that at least one shows a spuriously favourable result grows with the count.

Switching this on applies a Bonferroni correction: α is divided by the number of ICCs, so each individual test must clear a higher bar.

Result: wider CIs and lower per-ICC power, but the confidence level now applies to the joint claim.

Notes & methodology

k vs. M

k is the calibration sample — outputs double-rated before main grading begins to verify the rubric is reliable. M is the sample size for the actual ICC estimate in the main study, drawn separately.

Calibration vs. CI-based power

The calibration check uses the point estimate: pass if observed r ≥ 0.70, relaxed band if 0.65 ≤ r < 0.70. The "Power vs ρ₀" metric is more conservative — it uses the CI lower bound: pass if the lower bound exceeds ρ₀. Both derive from the same Fisher-z sampling distribution but answer different questions.

How the recommendations are derived

k is the smallest size giving ≥ 80% probability of strict pass at the assumed ρ. If ρ ≤ strict threshold, calibration cannot reliably pass and the recommendation flags the underlying issue.

M is the larger of (a) Bonett's N for ±0.10 CI half-width and (b) the N for ≥ 80% power that the CI lower bound exceeds ρ₀. The binding criterion is noted.

Data-generating model

Each output is drawn as y_ij = μ + s·(α_i + e_ij) with Var(α) = ρ, Var(e) = 1 − ρ. The latent score is shifted to the centre of the rating scale, scaled so ±2 SD covers the range, then rounded and clipped to the scale bounds.

Estimator and CI

ICC is computed as (MS_between − MS_within) / (MS_between + MS_within) — one-way random-effects, ICC(1,1). CI uses Fisher-z: W = ½·log((1+r)/(1−r)), SE(W) ≈ 1/√(n−2) · √((Pop−n)/(Pop−1)).

Multiple ICCs

With 3 rater pairs × D dimensions, you'll report 3D separate ICCs in the main study. Joint-confidence mode applies Bonferroni: divides α by 3D, so each individual ICC must clear a higher bar.
